# 1. 現状の課題 / 현재의 과제

## 業務上の問題点 / 업무상의 문제점

| 課題 / 과제 | 影響 / 영향 |
|-------------|-------------|
| Excelでの在庫管理 → 入力ミス、重複 | 出荷遅延、在庫不一致 |
| 各運送会社サイトに個別ログイン | 時間浪費、パスワード管理の煩雑さ |

# 3. 業務フロー変革 / 업무 플로우 변화

## Before (従来) / 이전

1. 注文受信 → メールを確認 → 手動で在庫チェック → Excelに記入
2. 送り状作成 → ヤマトのサイトにログイン → 手入力 → 印刷
3. 在庫更新 → Excelを手動更新 → 共有ドライブにアップロード
4. 情報共有 → 口頭、メモ、LINE

## After (新システム) / 이후

1. 注文受信 → **自動取込** → ダッシュボードに表示 → ワンクリック承認
2. 送り状作成 → InvenTree**プラグインが自動作成** → 印刷ボタンのみ
3. 在庫更新 → **リアルタイム自動更新** → 全員が同じデータを確認
4. 情報共有 → **Wiki + タスク管理** → 検索可能、履歴あり

# 4. 主要機能 / 주요 기능

## 統合ダッシュボード / 통합 대시보드

- Web/Android両対応

# 5. AI業務支援 / AI 업무 지원

## AIアシスタントの役割 / AI 어시스턴트의 역할

**操作ガイド** — 画面を見ながらステップバイステップで指示
- 「この注文はどう処理すればいいですか？」
- 「送り状の印刷方法を教えてください」

**タスク管理** — 自然言語でタスク操作
- 「来週のヤマト集荷をタスクに追加して」
- 「佐川の未完了タスクを見せて」

**会議記録** — 音声から自動でタスク抽出
- 会議録音 → 文字起こし → タスク候補抽出 → 承認後にVikunjaに反映

**個人情報保護**: 全てのデータはAIに送る前にマスク処理されます

---

# 6. セキュリティ / 보안

## データ保護の仕組み / 데이터 보호 구조

| 項目 / 항목 | 対策 / 대책 |
|-------------|-------------|
| 認証 | Google SSO（シングルサインオン） |
| 通信 | 全て暗号化（TLS/HTTPS） |
| AI個人情報 | PII自動マスク処理後にのみAI送信 |
| 生データ保持 | 7日間のみ、自動削除 |
| バックアップ | 暗号化＋GCS自動アップロード、毎日 |
| アクセス制御 | 社員Googleアカウントのみ |

- 外部クラウドにはバックアップとAI処理のみ
- 業務データは社内サーバーに保管
