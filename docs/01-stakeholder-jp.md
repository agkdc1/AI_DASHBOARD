# シンビジャパン 統合業務システム

**Shinbee Japan Integrated Business System**

2026年2月

---

# 1. 現状の課題

## 業務上の問題点

| 課題 | 影響 |
|------|------|
| Excelでの在庫管理 → 入力ミス、重複 | 出荷遅延、在庫不一致 |
| 各運送会社サイトに個別ログイン | 時間浪費、パスワード管理の煩雑さ |
| 社内知識が個人に依存 | 退職時に業務知識が失われる |
| タスク管理が口頭ベース | 漏れ、重複作業 |
| マーケットプレイス注文の手動処理 | 処理遅延、エラー |

# 2. システム概要

```
              インターネット
                     |
              [ルーター]
                     |
         +-------------------+
         | Raspberry Pi      |  ← 制御装置
         | + FAXシステム     |
         | + TCP プロキシ    |
         +-------------------+
          Tailscale VPN ネットワーク
              /         \
     +-----------+  +-----------+
     | PC-1      |  | PC-2      |  ← 業務サーバー
     | 在庫管理  |  | 自動化    |
     | タスク管理 |  | 送り状印刷|
     | Wiki      |  | AI支援    |
     +-----------+  +-----------+
```

**全てのデータは社内に保管**

---

# 3. 業務フロー変革

## Before（従来）

1. 注文受信 → メールを確認 → 手動で在庫チェック → Excelに記入
2. 送り状作成 → ヤマトのサイトにログイン → 手入力 → 印刷
3. 在庫更新 → Excelを手動更新 → 共有ドライブにアップロード
4. 情報共有 → 口頭、メモ、LINE

## After（新システム）

1. 注文受信 → **自動取込** → ダッシュボードに表示 → ワンクリック承認
2. 送り状作成 → InvenTree**プラグインが自動作成** → 印刷ボタンのみ
3. 在庫更新 → **リアルタイム自動更新** → 全員が同じデータを確認
4. 情報共有 → **Wiki + タスク管理** → 検索可能、履歴あり

# 4. 主要機能

## 統合ダッシュボード

**4つのタブで全業務をカバー:**

| タブ | 機能 | 連携先 |
|------|------|--------|
| 在庫 | 部品、在庫、注文管理 | InvenTree |
| タスク | カンバン、カレンダー、担当割り当て | Vikunja |
| Wiki | 社内マニュアル、手順書 | Outline |
| 設定 | 言語切替 (日/韓/英)、テーマ | — |

- Googleアカウントで一度ログインするだけ
- Web/Android両対応

# 5. AI業務支援

## AIアシスタントの役割

**操作ガイド** — 画面を見ながらステップバイステップで指示
- 「この注文はどう処理すればいいですか？」
- 「送り状の印刷方法を教えてください」

**タスク管理** — 自然言語でタスク操作
- 「来週のヤマト集荷をタスクに追加して」
- 「佐川の未完了タスクを見せて」

**会議記録** — 音声から自動でタスク抽出
- 会議録音 → 文字起こし → タスク候補抽出 → 承認後にVikunjaに反映

**個人情報保護**: 全てのデータはAIに送る前にマスク処理されます

---

# 6. セキュリティ

## データ保護の仕組み

| 項目 | 対策 |
|------|------|
| 認証 | Google SSO（シングルサインオン） |
| 通信 | 全て暗号化（TLS/HTTPS） |
| AI個人情報 | PII自動マスク処理後にのみAI送信 |
| 生データ保持 | 7日間のみ、自動削除 |
| バックアップ | 暗号化＋GCS自動アップロード、毎日 |
| アクセス制御 | 社員Googleアカウントのみ |

- 外部クラウドにはバックアップとAI処理のみ
- 業務データは社内サーバーに保管
