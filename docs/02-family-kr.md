# 프로젝트 소개

## 신비재팬에 무엇이 필요했나

회사가 매일 하는 일:
- 라쿠텐, 아마존 등에서 들어오는 주문 처리
- 야마토, 사가와 택배 운송장 출력
- 재고 관리 (지금까지 엑셀로...)
- 팩스 송수신 (일본은 아직도 팩스를 많이 씀)
- 사내 업무 정보 공유

**문제**: 전부 수동, 사이트마다 따로 로그인, 실수 많음

**해결**: 하나의 시스템으로 전부 통합!

# 얼마나 큰 프로젝트인가

## 라즈베리 파이 하나로 시작

라즈베리 파이 = 손바닥 크기의 컴퓨터 (5만원)

거기에 노트북 2대를 연결해서:

- **팩스 서버** (NTT 광회선 직접 연결)
- **재고관리 시스템** (InvenTree)
- **업무관리 시스템** (Vikunja)
- **사내 위키** (Outline)
- **통합 대시보드** (Flutter 앱)
- **AI 어시스턴트** (Gemini)
- **자동화 봇** (택배 사이트 자동 조작)

대기업이 수천만원 들여서 만드는 걸, 라즈베리 파이 + 노트북으로 구현

# 이전 vs 이후

## 매일의 변화

### 이전
```
주문 들어옴 → 메일 확인 → 엑셀에서 재고 찾기 → 야마토 사이트 로그인
→ 주소 직접 입력 → 운송장 출력 → 엑셀에서 재고 수정 → 파일 공유
```
**소요시간: 주문 1건당 15-20분**

### 이후
```
주문 들어옴 → 대시보드에 자동 표시 → 확인 버튼 클릭
→ 운송장 자동 생성 → 출력 → 재고 자동 업데이트
```
**소요시간: 주문 1건당 2-3분**

# 어려웠던 점

## 기술적 도전

**1. NTT 팩스 연결**
- 일본 NTT 광회선의 SIP 프로토콜이 표준과 다름
- 인증이 IP 기반 (비밀번호 없음)
- DTMF 신호가 오디오 내장 (RFC 2833이 아님)
- 라즈베리 파이에서 MAC 주소를 스푸핑해서 해결

**2. WiFi 노트북으로 쿠버네티스**
- 일반적으로 K8s는 고정 IP 서버에서 운영
- WiFi + VPN으로 동작하게 만듦 (보통은 안 함)
- 5분마다 IP 주소 자동 갱신

**3. ARM에서 Windows 설치파일 빌드**
- 라즈베리 파이(ARM)에서 Windows 프로그램 빌드
- QEMU가 안 되서 box86으로 해결 (전 세계적으로 거의 안 하는 방법)
